Server/Workstation Alerts

Disk Space:

  • NOC Technician will change ticket status from ‘New’ to ‘NOC Working Issue Now’.
  • Run disk clean up on the affected drive.
  • Delete any unneeded files from the htadmin user’s documents and downloads.
  • Download and run TreeSize to determine what is taking up space.
  • NOC Technician will determine what can be deleted based on TreeSize report.
  • NOC Technician will escalate to Market Engineering to determine if space can or needs to be added.

CPU/Memory Utilization:

  • NOC Technician will change ticket status from ‘New’ to ‘NOC Working Issue Now’.
  • NOC Technician will evaluate and determine what process is taking up the resources and if it can be stopped/restarted.
  • If the issue is a repeated occurrence (3 or more time within a week) NOC Technician will check if AV or EDR is root cause.
  • If so, put in an exception.
  • NOC Technician will escalate to Market Engineering to determine if work needs done on the process that is causing high utilization or if the threshold needs increased on the alert.
  • If threshold needs increased NOC Techician will increase to an appropriate amount and put in the devices Notes the CW ticket # and what was adjusted.
  • If threshold has already been increased to 99% a discussion with Engineering and NOC will need to take place to ensure health of the server is alright and/or if work needs to be done.

User Added to Privileged Group

  • NOC Technician will change ticket status from ‘New’ to ‘NOC Working Issue Now’.
  • NOC Technician will evaluate information from alert to determine if user was legitimately added or not.
  • If htadmin/htms or client, IT admin has added the user this is more then likely a legitimate action.
  • If this has been determined to be unauthorized
    • Disable user account.
    • Notify Jason F. and Andrew K.
    • Call POC of client (idk who will do that probably a manager)

Patching

  • NOC Technician will change ticket status from ‘New’ to ‘NOC Working Issue Now’.
  • Work patching issue as normal to get patch installed.
  • Once patch is installed let Ninja go through its patch scan, so it recognizes the patch has been installed.
  • NOC Technician will change ticket status from ‘NOC Working Issue Now’ to ‘Closed’.
  • NOC Technician will reset the condition in Ninja on the endpoint.

Network Alerts

CPU/Memory Utilization:

  • NOC Technician will change ticket status from ‘New’ to ‘NOC Working Issue Now’.
  • If the issue is a repeated occurrence (3 or more time within a week) NOC Technician will escalate to Market Engineering to determine if the device needs replaced or if the alert can be suppressed.
  • NOC Technician will change the devices policy to the <device> Up Down policy to only alert on up down.

NMS Agent/Server Down:

  • NOC Technician will bundle the NMS Agent ticket to the Server down ticket with the server down ticket being the parent ticket.
  • NOC Technician will change ticket status from ‘New’ to ‘NOC Working Issue Now’.
  • NOC Technician will attempt to get the server back online.
  • NOC Technician will escalate to Market Engineering if onsite is required to bring server back online.

VMWare Alerts

Datastore Utilization:

  • NOC Techician will change ticket status from ‘New’ to ‘NOC Working Issue Now’.
  • Make sure no snapshots are present.
  • Escalate to TS to have them check if anything can be cleaned up.
  • If not raise threshold in the alert.

Snapshot Lifespan

  • NOC Technician will escalate to Market Engineering to determine if snap can be deleted or if it’s still needed.
